# Claude.md

このプロジェクトに関する Claude 向けのメモ・指示を記載します。

---

# プロジェクト概要

- カフェのウェブサイト（静的サイト）
- 将来的にWordPress化しやすい構成を前提とする
- GitHub Pagesで公開することを前提とする

---

# フォルダ構成

- `index.html`
  - トップページ

- `assets/`
  - css
  - js
  - img

- `assets/css/`
  - reset.css
  - style.css
  - style.css.map
  - style.scss

- `assets/js/`
  - main.js

- `parts/`
  - 共通パーツ（header, footer, drawer）

- `favicon/`
  - ファビコン

---

# 回答ルール

- 日本語で回答する
- 初心者にも分かるように説明する
- 修正理由を説明する
- 変更箇所を明確にする
- コピペできる完成コードを出力する
- 省略せずに出力する
- 必要に応じてコードコメントを追加する

---

# 禁止事項

- 既存のclass名を勝手に変更しない
- 指示されていないライブラリを追加しない
- ファイル構成を変更しない
- 必要以上に既存コードを書き換えない
- 推測で実装しない
- デザインを大きく変更しない
- JavaScriptライブラリを勝手に導入しない

---

# 使用を許可する外部リソース

- Google Fonts（Shippori Mincho / Parisienne）を使用してよい
- フォントは `<link>` でCDNから読み込む（自前設置はしない）
- 上記以外のライブラリ・CDNは無断で追加しない

---

# フォルダ構成ルール

- WordPress化しやすい構成を維持する
- assets配下にcss/js/imgを格納する
- partsフォルダでパーツ管理する
- セクション単位で保守しやすく構築する
- 相対パスで記述する
- ファイル名は小文字+kebab-caseを使用する

---

# HTMLルール

- セマンティックHTMLを使用する
- BEM命名を使用する
- divの多用を避ける
- 適切なalt属性を設定する
- 見出し階層を正しく使用する
- インデントを揃える
- 再利用しやすい構造にする

---

# CSSルール

- Sassを使用する
- style.scssでスタイルを管理する
- コンパイル後のstyle.cssを読み込む
- reset.cssを使用する／先に読み込む
- 保守しやすいCSSを書く
- レスポンシブ前提で作成する
- PCファーストで実装する
- 1024px以下で調整する
- 横スクロールを発生させない
- pxを優先的に使用する
- 必要に応じて%やvwを使用する
- 過度な!importantは使用しない

---

# JavaScriptルール

- 普通のJavaScriptを使用する
- main.jsを基本とする
- 必要に応じて機能ごとに分割する
- 必要最小限の実装にする
- 可読性を重視する
- 複雑な処理はコメントを追加する
- class操作を中心に実装する
- jQueryや外部ライブラリは必要時のみ使用する

---

# デザインルール

- シンプルで洗練されたデザイン
- 余白を広めに取る
- Instagram風の雰囲気
- 落ち着いた色味
- カフェサイトのようなおしゃれ感
- 高級感を意識する
- モバイルでも見やすくする

---

# コード出力ルール

- 修正後の完成コードを出力する
- 修正箇所を説明する
- HTML/CSS/JavaScriptを分けて出力する
- 追加位置を明確に説明する
- 不要なコードは増やさない

---

# パスルール

- `./` から始まる相対パスを使用する（assets / parts / favicon すべて対象）
- GitHub Pagesで動作する構成を前提とする
- ローカル環境依存の記述を避ける

---

# 将来的なWordPress化を意識する

- header/footerなどをパーツ化しやすく構築する
- セクション単位で分割しやすい構造にする
- 再利用性を意識する
- functions.php移行を想定した構成を意識する
- WordPressテーマ化しやすいclass設計にする

---

# デザインの方向性

- おしゃれで洗練されたデザインを重視する
- カフェ・美容室・店舗サイト風の雰囲気を意識する
- 写真を活かしたレイアウトにする
- 余白・文字組み・視線誘導を意識する
- シンプルだが安っぽく見えないデザインにする
